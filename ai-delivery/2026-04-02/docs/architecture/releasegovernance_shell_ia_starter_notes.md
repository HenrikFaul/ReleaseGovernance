# ReleaseGovernance shell IA starter notes

## Why this package is needed
The current system already has a working shell, but the information architecture is still too dashboard-heavy and not workflow-first enough.

## What this starter introduces
- separate global navigation model
- separate project navigation model
- new route grouping idea under a workspace layer
- first-class route directions for portfolio, overview, parity, traceability and automation

## Important migration rule
Current working routes must remain reachable during the transition.
The new structure should be introduced scaffold-first, not as destructive replacement.
