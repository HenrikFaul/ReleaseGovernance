# ReleaseGovernance development prompt - v1.0.3 / 26040601

## Kötelező előfeltétel
1. Olvasd végig az `AI_DEVELOPER_PROMPTING_MASTER_MANIFESTO.md` fájlt.
2. Olvasd végig a `design-master-rules.md` fájlt.
3. Olvasd végig a `CHANGELOG.md`, `codingLessonsLearnt.md` és `codingLessonsLearnt.local.md` releváns részeit.
4. Ne bontsd meg a már működő Import / Push to Jira / listanézet / mobil navigáció működést.

## Megértett üzleti igény
- A release oldal maradjon Vercel-szerű lista fókuszú.
- Új commit vagy deploy után a detektált release candidate ténylegesen jelenjen meg a release dashboardon.
- A release candidate legyen jóváhagyható governed release-szé.
- A Jira backfill CSV export ne legyen üres, ha unresolved backfill candidate-ek léteznek.
- A projekt dashboard legyen három világos csoportra bontva: Release dashboard, Capabilities dashboard, Integrations dashboard.
- A sidebarból tűnjenek el a külön Releases / Capabilities / Integrations gombok; maradjon egy project-szintű dashboard entry.
- A header közepén az aktív projekt neve jelenjen meg.
- Mobilon maradjon közvetlen Project gyorsgomb.
- A runtime létrehozott projektek és importált/jóváhagyott release-ek ugyanúgy működjenek a részletes oldalakon is, ne csak seed adatokkal.
